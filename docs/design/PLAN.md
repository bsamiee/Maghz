# [MASTER_REALIZE_PLAN]

The eight per-domain worklists fold into one realize campaign. The implement pass executes domains in the topological order below: every domain's owners exist before any consumer composes them, and every canonical shape is realized by its owning domain before a downstream domain reads it. Each worklist remains the decision-complete spec for its own owners, ADTs, `.api` members, and acceptance; this plan owns only the cross-domain sequencing, the deduplicated dependency admission, and the canonical-shape ownership map.

## [01]-[TOPOLOGICAL_REALIZE_ORDER]

A total order over the eight domain keys honoring every `depends_on`. `settings`, `infra`, `db`, and `ledger` are not separate domains — they resolve to owner files inside `existing-rails` (`admin/settings/config.py`, `admin/infra/`, `admin/db.py`, `admin/rails/ledger.py`). `secrets`/`secrets-bootstrap` is an external Forge/`setup-env.sh`/1Password bootstrap, not a Python-owner node; it supplies env values for table rows and never blocks a Python owner.

| [N] | [DOMAIN]         | [DEPENDS_ON (resolved)]                         | [UNBLOCKS]                                              |
| :-: | ---------------- | ---------------------------------------------- | ------------------------------------------------------ |
|  1  | `runtime`        | —                                              | every other domain (the substrate)                     |
|  2  | `existing-rails` | — (after `runtime` per substrate law)          | `mcp`, `cloud-sync`, `n8n`, `remote`, `automation`     |
|  3  | `mcp`            | `existing-rails`, `runtime`                    | `n8n`, `integrations`, `automation`                    |
|  4  | `cloud-sync`     | `runtime`, `existing-rails`                    | `remote`, `automation`                                 |
|  5  | `n8n`            | `runtime`, `existing-rails` (infra), `mcp`     | `automation`                                           |
|  6  | `remote`         | `runtime`, `existing-rails`, `cloud-sync`      | (leaf — VPS exec/deploy consumer)                      |
|  7  | `integrations`   | `runtime`, `mcp`                               | `automation`                                           |
|  8  | `automation`     | `runtime`, `existing-rails`, `mcp`, `integrations`, `n8n` | (top-level orchestrator — leaf)             |

Cluster annotation:

- **Runtime substrate first (1).** `admin/runtime/` (`rails`/`lanes`/`resilience`/`receipts` + `__init__`) is realized as a standalone package with zero domain prerequisites. It adds NO coupling to `admin/rails/*` during its own pass; the rails rewiring is deferred to the `existing-rails` pass, which composes the already-landed `guard`/`async_boundary`. This is why `runtime` precedes `existing-rails` even though `existing-rails` owns the canonical wire shapes: the substrate package must exist before its rewiring is applied.
- **Existing rails after runtime (2).** `existing-rails` restructures `admin/rails/*` in one pass: deletes `admin/rails/stack.py` (relocating `StackOp`/`StackDetail`/`_BUILD`/Pulumi dispatch into `admin/infra/runner.py` as `run(op: StackOp, cfg)`), collapses `schema` apply/doctor into `run(op: SchemaOp, cfg)`, collapses `sync` diff/generate into `run(cfg,/,*,concept)` (no `SyncOp`), and in the SAME pass supersedes its two `@stamina.retry` bootstrap sites and local `retry_boundary` aspect with runtime's `guard(RetryClass.*)`. It establishes every canonical wire shape (`Envelope`/`Detail`/`Status`, `MaghzSettings`, `DbFault`/`Boundary`, the root `app`).
- **MCP after settings + runtime (3).** Composes `MaghzSettings` (adds `McpServerSettings` nested group) and the `admin.core` `Envelope`/`Detail`/`Status` substrate; owns `_SERVER_TABLE` and the generated `.mcp.json`.
- **Cloud-sync after runtime + rails (4).** Composes `LanePolicy.drain`/`DrainReceipt`/`ContentKey`/`RetryClass.PROC` and the shared `Envelope`+`Detail` contract; owns the `frozendict[Remote,RemoteConfig]` settings table.
- **N8N after infra + mcp (5).** Composes the Pulumi program in `admin/infra/stack.py` (adds the n8n container/volume) and the MCP `N8N_API_URL == cfg.n8n.api_url` contract; the n8n Python surface has zero hard code-coupling to mcp (only the `_SERVER_TABLE[ServerKind.N8N]` row equality requires mcp realized).
- **Remote after cloud-sync (6).** `deploy(target, StackOp.UP)` invokes `cloud restore` over SSH; imports the relocated `StackOp`/`StackDetail` (from `admin/infra/runner.py`) and `SchemaDetail` read-only; extends runtime's `CLASSIFY` with asyncssh rows in place.
- **Integrations after mcp (7).** `IntegrationsConfig` declares the WORKSPACE-row requirements that mcp's `_SERVER_TABLE` owns; the agy shim composes the runtime resilience band.
- **Automation last (8).** The top orchestrator composes runtime's `drain`/`guard`/`Receipt`, `existing-rails` `sync.run`, mcp+integrations `_AGENT_DISPATCH` callables, and n8n (future `AgentSkill.N8N_TRIGGER`); reads the `ledger` `Kind.OWNER` projection as a soft read-view.

Admission notes:

- **`frozendict` is the single admission resolving the cross-domain table-form tension.** `existing-rails` §09 acceptance forbids `MappingProxyType` for its four dispatch/correspondence tables (`_RANK_EXIT`/`_SQL`/`_BUILD`/`_RUNNER`) and `cloud-sync` requires `frozendict[Remote,RemoteConfig]` at the pydantic-settings boundary (`expression.Map` does not serve a typed settings table). One admission satisfies both. It is NOT converged onto the other domains' tables: `runtime`, `n8n`, and `automation` keep `expression.Map`/`MappingProxyType` per their own worklists (see ripple R-FROZENDICT). `mcp` and `remote` list `frozendict` only as a *fallback*; once admitted here they use it for `ServerSpec.env`/`docker_env` and `_REMOTE_ENV` rather than `MappingProxyType`, but neither admission is theirs to make. Import `frozendict.frozendict` explicitly until py3.15 stdlib promotion.
- **`asyncssh`** is a pure-Python wheel; `uv lock` regenerates cleanly. License EPL-2.0 OR GPL-2.0+ (unmodified library). If stubs are absent, extend the `pg8000`-pattern per-package mypy override to `asyncssh`. The `.api` slice is domain-local (`admin/remote/.api/asyncssh.md`), authored before any production source.
- **`apscheduler` must be the 4.x async-first surface.** Remove any `apscheduler<4.0` pin; the 3.x `AsyncIOScheduler`+`AsyncIOExecutor` path is asyncio-only and violates the anyio mandate. The `.api` catalog is a full 4.x rebuild (`AsyncScheduler`, `CronTrigger.from_crontab`, `ConflictPolicy`, `add_schedule`, `run_until_stopped`, `subscribe`, `JobReleased`, `JobOutcome.success/.missed`).
- **CLI-band tools** (`agy`/antigravity-cli, `workspace-mcp`, `codex-plugin-cc`, Node.js 18.18+, `rclone` v1.74.3+) are Forge-provisioned on `PATH`, subprocess/plugin boundaries only — no `pyproject.toml` change. Their evidence slices are skill-local (`.claude/skills/agy/workspace-mcp.api.md`, `codex/SKILL.md`) or `docs/.api/rclone.md`, authored at the consuming domain's realize.
- **`.api` directory layout.** No `.api` directory exists yet. `docs/.api/` is the repo-shared catalog root for library surfaces (`frozendict`/`watchfiles`/`apscheduler`/`rclone`); `admin/remote/.api/` is the domain-local slice for `asyncssh`. The owning domain authors each catalog BEFORE realizing the code that composes it.

## [03]-[RIPPLE_GRAPH]

Each row is a canonical shape, its sole owning domain, and the downstream domains that compose it read-only. The implement pass realizes the OWNER column first; every CONSUMER composes the named shape and never forks a parallel surface. Counterpart shapes that look analogous but are intentionally distinct are flagged.

| [ID]            | [CANONICAL SHAPE]                                                                 | [OWNER]          | [CONSUMERS]                                  |
| --------------- | -------------------------------------------------------------------------------- | ---------------- | -------------------------------------------- |

## [04]-[REALIZE_NOTES_PER_DOMAIN]

- **runtime** — Realize `admin/runtime/` (`rails`/`lanes`/`resilience`/`receipts`/`__init__`) as the standalone substrate: four closed unions (`BoundaryFault`/`Admit`/`Receipt` + `RetryClass`/`RetryMode`), `CLASSIFY` as the sole exception→fault contract, zero new deps (all seven packages already admitted), `expression.Map` for every keyed table; defer the `admin/rails/*` rewiring to the existing-rails pass.
- **existing-rails** — Restructure `admin/rails/*` in one pass: delete `stack.py` (relocate `StackOp` to `admin/infra/runner.py`), collapse `schema`→`run(SchemaOp)` and `sync`→`run(cfg,concept=)`, drop the `schema_apply`/`schema_doctor`/`sync_diff`/`sync_generate` alias family for one alias per rail, admit `frozendict` for the four dispatch tables, replace the manual `anyio.run` trampoline with `App.run_async(backend='asyncio')`, and supersede the two `@stamina.retry` bootstrap sites with runtime's `guard`.
- **mcp** — Create `admin/mcp/` (`model.py` = `ServerKind`/`McpOp`/`ServerSpec`/`McpConfigDetail`/`_SERVER_TABLE`; `ops.py` = `McpFault` + `mcp(op,cfg)`), add `McpServerSettings` nested group to `MaghzSettings`, regenerate `.mcp.json` (7 keys, indent=2, `${MAGHZ_MCP__*}` placeholders) as the committed artifact; no new packages.
- **cloud-sync** — Create `admin/rails/cloud.py` as the single backup+restore owner (`CloudOp`/`Remote`/`CloudSyncDetail`/`CloudFault` + `run`), add `RemoteConfig`+`CloudConfig` with the `frozendict[Remote,RemoteConfig]` validator, compose `LanePolicy.drain`+`Admit.retried(RetryClass.PROC)` for rclone fan-out, author `docs/.api/rclone.md`; mount under the `cloud` sub-app to avoid the `sync` namespace.
- **n8n** — Create `admin/rails/n8n.py` (`N8nOp`/`N8nDetail`/`run` over export/import/status using `MappingProxyType` `_BUILD`, not `expression.Map`), add `N8nConfig` with `api_url` computed field, extend `admin/infra/stack.py` `define()` with the n8n container/volume threading the colima provider and binding `db_container` for `depends_on`; zero new packages.
- **remote** — Create `admin/remote/` (`connection.py` = asyncssh lifecycle + `KnownHostsPolicy` + `RemoteTarget`; `ops.py` = `exec`/`deploy` + `_push_tree` + `ExecReceipt`/`DeployReceipt`), add `RemoteConfig` subgroup, admit `asyncssh>=2.23.1`, author `admin/remote/.api/asyncssh.md` before source, extend runtime's `CLASSIFY` with asyncssh rows in place; no `RemoteFault`, no second SSH client.
- **integrations** — Add `IntegrationsConfig` to `MaghzSettings`; realize `.claude/skills/agy/scripts/agy.py` as the only Python shim (`AgyOp`/`AgyFault`/`AgyReceipt`/`agy(op,*,args)` over `anyio.run_process` inside `move_on_after` + `stamina.retry`); author `.claude/skills/agy/workspace-mcp.api.md` and the agy/codex `SKILL.md`s; no `pyproject` change.
- **automation** — Create `admin/automation/` (`model.py` = `Trigger`/`Action`/`AgentSkill`/`AutomationSpec`/`AutomationReceipt`/`AutomationFault`; `engine.py` = `drive(spec,cfg)` with anyio task-group lanes, governor aspect, total `_exec` dispatch, `expression.Map` `_AGENT_DISPATCH`/`_LANE_POLICIES`), add `AutomationConfig`, admit `watchfiles`+`apscheduler>=4.0.0` (4.x async-first), author both `.api` catalogs before realize; the engine dispatches one action per trigger cycle (sequencing is n8n's domain).
