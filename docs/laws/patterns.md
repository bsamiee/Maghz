# [PATTERNS]

Pattern law binding two or more execution surfaces. A row admitted here binds every surface it names; a single-surface law routes to that surface's stack doctrine or the `docs/standards/` pair instead.

## [01]-[ROWS]

| [INDEX] | [LAW]                                                                                         | [BINDS]                              |
| :-----: | :-------------------------------------------------------------------------------------------- | :----------------------------------- |
|  [01]   | a multi-surface fact declares once in a catalog owner and renders as `[CATALOG:<tag>]` blocks | Python, SQL, Dockerfile              |
|  [02]   | a repo-owned path projects from the module-resolved `REPO_ROOT`, never cwd                    | rails, SQL apply, image, automations |

A marked block regenerates through its catalog owner, never by hand edit. `admin/settings.py` mints `REPO_ROOT` and the `AnchoredPath` validator that enforces the path law.
